VidaaEdge - Self-hosted

Install custom apps on your VidaaOS-based TV with this easy-to-use self-hosted tool.

Prerequisites

Vidaa OS TV

Ensure you have a compatible TV or device running Vidaa OS.

Node.js

Version 18.19.1 or newer (for Angular 19)

NPM or Yarn

Use NPM (included with Node.js) or Yarn to manage packages.

3. Start the Development Server

Use the following command to start the development server. The dev server is configured with a self-signed certificate, exposed to the local network, and SSL is enabled by default:

4. Set DNS Server

Configure your DNS settings to map vidaahub.com to your local or server IP address. This step is essential for the app to function correctly. You can achieve this in one of the following ways:

  • Option 1: Use a Dedicated DNS Server
    Set up a custom DNS server (e.g., using BIND or Pi-hole) or any other and configure it to resolve vidaahub.com to your desired local or server IP address.
  • Option 2 (easy): Configure DNS Directly in Your Router
    If your router supports custom DNS settings, you can configure it to map vidaahub.com to your local server IP address. The steps may vary depending on your router model, but the general process is as follows:
    1. Log in to your router's admin panel (e.g., http://192.168.xxx.1).
    2. Navigate to the DNS or "Static Hostname" settings section.
    3. Add a rule to resolve vidaahub.com to your local or server IP address (e.g., 192.168.xxx.123).
    4. Save the configuration and restart your router if necessary.

5. Open the App on your TV

Set up your Vidaa OS TV to use the DNS server you configured in the previous step (if you used a router, this step is not necessary). Open your browser and navigate to https://vidaahub.com to access the app!

6: Install/Uninstall Apps

See the Installation section for more informations.

Important Note

A self-signed certificate has been configured for the development server. This is sufficient to utilize the global functions of Vidaa OS (Hisense API).

If you intend to deploy this project in a production environment, ensure the following steps are completed:

  • Build the project for production.
  • Provide a DNS record that users can point their TVs to, so the project is accessible under vidaahub.com.
  • Set up SSL and HTTPS. These are mandatory for the Vidaa OS global functions to work.

Detailed instructions for production deployment are omitted here, as it is expected that you are familiar with the requirements for configuring DNS, SSL, and hosting a project.

Debug

In development mode, the debug mode is enabled by default. Check the environment files to change this setting. The debug mode skips VidaaOS-based device detection for faster testing.
